# 日记 工作流

## 依赖 Skill

| Skill | 用途 |
|-------|------|
| `obsidian` | 读取前一天日记、读取模板、写入正式日记 |
| `apple-reminders` | 记录中包含提醒事项时调用 |
| `apple-notes` | 记录中包含备忘录时调用 |
| `caldav-calendar` | 记录中包含日程时调用 |

## 环境变量

| 变量 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `DIARY_VAULT` | 否 | - | Obsidian vault 名称 |
| `DIARY_DIR` | 否 | `Diary` | 日记在 vault 内的目录名 |
| `DIARY_TEMPLATE` | 否 | `Template/Diary.md` | 日记模板在 vault 内的路径 |

## 临时记忆文件

碎片内容实时追加到 OpenClaw 当天的记忆 `memory/yyyy-mm-dd.md`

---

## 场景一：实时记录

**触发条件：** 用户消息包含 `日记` 等关键字，或显式调用 `/diary`。

**处理流程：**

1. 理解并整理用户的日记内容（去除语气词，提炼关键信息，保持原意）
2. 以 Markdown bullet 格式追加写入 `memory/yyyy-mm-dd.md`，附带时间戳
3. 如果内容中包含以下意图，同步处理：
   - **日程**（"下午三点开会"、"明天约了 X"）→ 调用 `caldav-calendar` 创建日历事件
   - **提醒**（"记得买 X"、"提醒我 X"）→ 调用 `apple-reminders` 添加提醒
   - **备忘**（"记一下 X"、"备忘：X"）→ 调用 `apple-notes` 创建备忘录

**临时文件格式示例：**

```markdown
# 2026-03-21

- 09:15 和产品开了需求对齐会，确定了下个版本优先做搜索优化
- 11:30 午饭前处理了三个 PR review
- 下午去健身房，腿部训练
```

## 场景二：生成正式日记

**触发条件：** 用户发送 `生成日记`，或定时任务触发。

**特别提醒：** 用户或者定时任务可能在凌晨执行，这种情况下需要生成前一天的日记。

**处理流程：**

1. 读取今天的临时记忆文件 `memory/yyyy-mm-dd.md`
2. 通过 `obsidian` 读取 `$DIARY_DIR/yyyy-mm-dd.md`（前一天的日记），作为上下文参考：
   - 保持写作风格和叙事连贯性
   - 关注昨天未完成的事项，在今天日记中自然跟进
3. 通过 `obsidian` 读取 `$DIARY_TEMPLATE` 模板文件，按模板结构生成日记
4. 将生成的日记通过 `obsidian` 写入 `$DIARY_DIR/yyyy-mm-dd.md`

**注意事项：**
- 日记应自然流畅，不要机械罗列条目
- 参考前一天风格，保持语气一致，避免前后割裂感
- 若临时文件不存在或为空，告知用户今天暂无记录，询问是否仍要生成
- 写入前检查 Obsidian 中今天的日记是否已存在，如果已经存在则追加写，并且提示用户
